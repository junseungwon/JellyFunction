# 젤리 캐릭터 셋업 가이드 — SkinnedMesh 방식 (리깅 캐릭터)

이 가이드는 **리깅된 캐릭터(SkinnedMeshRenderer)** 로 젤리 캐릭터를 만들 때만 기준으로 합니다.

---

## ▼ Hierarchy에서 설정하는 순서 (SkinnedMesh 기준)

아래 순서대로 하면 됩니다.

---

### 1단계: 레이어 만들기 (한 번만)

1. 메뉴 **Edit → Project Settings** 열기  
2. 왼쪽에서 **Tags and Layers** 선택  
3. **Layers**에서 빈 **User Layer**에 다음 입력:
   - 예: User Layer 6 → `Ground`
   - 예: User Layer 7 → `Wall`

---

### 2단계: 오브젝트 만들기

1. **Hierarchy** 빈 곳 우클릭 → **Create Empty**
2. 생성된 오브젝트 이름을 **JellyCharacter**로 변경 (Inspector 상단 이름란)
3. **JellyCharacter** 선택한 상태에서 다시 우클릭 → **Create Empty**
4. 자식 이름을 **JellyBody**로 변경  

**최종 구조 (SkinnedMesh 방식):**
```
Hierarchy
└── JellyCharacter     ← 물리·입력·콜라이더
    └── JellyBody      ← SkinnedMeshRenderer + JellySkinnedMeshDeformer
```

JellyBody에는 **MeshFilter / MeshRenderer / JellyMeshDeformer는 넣지 않습니다.**  
리깅된 메시만 쓰므로 **Skinned Mesh Renderer**와 **JellySkinnedMeshDeformer**만 사용합니다.

---

### 3단계: JellyCharacter에 컴포넌트 넣기

**JellyCharacter**를 선택한 뒤 Inspector에서 아래를 **순서대로** 추가·설정합니다.

| 순서 | 할 일 | 방법 |
|------|--------|------|
| ① | **Rigidbody** 추가 | Add Component → 검색창에 `Rigidbody` 입력 → Rigidbody 추가 |
| ② | **Capsule Collider** 추가 | Add Component → `Capsule Collider` 추가 |
| ③ | **JellyPhysicsCore** 추가 | Add Component → 프로젝트 스크립트에서 `JellyPhysicsCore` 선택 |
| ④ | **JellyGapTraversal** 추가 | Add Component → `JellyGapTraversal` 선택 |
| ⑤ | **JellyCharacterController** 추가 | Add Component → `JellyCharacterController` 선택 |

**Rigidbody** 설정:

- Mass: `1`
- Drag: `0.5`
- Angular Drag: `10`
- Use Gravity: **체크**
- Freeze Rotation: **X, Z만 체크** (Y는 해제)

**Capsule Collider** 설정:

- Center: `0, 0.5, 0`
- Radius: `0.4`
- Height: `1`
- Direction: **Y-Axis**

**JellyPhysicsCore** 설정 (권장값):

- Elasticity: 8, Damping: 2, Max Compression: 0.3, Max Expansion: 1.8  
- Impact Deform Strength: 0.85, Impact Recovery Speed: 4  
- Movement Wobble Strength: 0.06, Wobble Frequency: 2  

**JellyGapTraversal** 설정:

- Gap Detection Range: 1.5, Detection Ray Count: 5  
- **Gap Wall Layer**: 드롭다운에서 **Wall** 선택  
- Min Passable Ratio: 0.25, Traversal Speed: 2  
- Adjust Collider On Squeeze: **체크**, Show Debug Gizmos: 원하면 체크  

**JellyCharacterController** 설정:

- Move Speed: 5, Jump Force: 8  
- **Ground Layer**: **Ground** 선택  

---

### 4단계: JellyBody 설정 (SkinnedMesh만)

**JellyBody**를 선택한 뒤 아래만 추가·설정합니다.  
**Mesh Filter / Mesh Renderer / JellyMeshDeformer는 추가하지 마세요.**

| 순서 | 할 일 | 방법 |
|------|--------|------|
| ① | **Skinned Mesh Renderer** 추가 | Add Component → `Skinned Mesh Renderer` |
| ② | **JellySkinnedMeshDeformer** 추가 | Add Component → `JellySkinnedMeshDeformer` |

**Transform** (JellyBody):

- Position: `0, 0.5, 0` (CapsuleCollider 중심과 맞추기)
- Rotation: `0, 0, 0`
- Scale: `1, 1, 1` 권장 (에셋 스케일 유지)

**Skinned Mesh Renderer**:

- **Mesh**: Project에서 리깅된 메시 에셋을 이 슬롯에 드래그
- **Root Bone**: 해당 스킨의 루트 본을 드래그 (보통 같은 프리팹/계층의 루트 본)
- **Bones**: 에셋 임포트 시 자동으로 채워져 있으면 그대로 둠
- **Materials → Element 0**: 사용할 머티리얼 할당 (URP Lit 등)

**JellySkinnedMeshDeformer**:

- **Deform Bones**:  
  - **비워 두면** Skinned Mesh Renderer의 **Root Bone**만 사용 (대부분 이렇게 사용)  
  - 변형을 줄 본을 더 넣고 싶으면 여기에 본을 추가 (예: Spine, Chest 등)
- Deform Multiplier: `1.5`
- Deform Smoothing: `0.08`
- Squash Stretch Ratio: `0.8`
- Preserve Volume: **체크**

---

### 5단계: 씬 오브젝트 레이어

- **바닥(지면)** 오브젝트 선택 → Inspector **Layer** → **Ground** 선택  
- **벽** 오브젝트 선택 → Layer → **Wall** 선택  

---

### 6단계: 확인 (SkinnedMesh 기준)

- **JellyCharacter**에 Mesh Filter / Mesh Renderer가 있으면 **제거**
- **JellyBody**가 JellyCharacter의 **자식 하나**로만 있는지 확인
- JellyBody에는 **Skinned Mesh Renderer**와 **JellySkinnedMeshDeformer**만 있고, MeshFilter·JellyMeshDeformer는 **없는지** 확인
- **Skinned Mesh Renderer**에 **Mesh**, **Root Bone**, **Materials**가 할당되어 있는지 확인
- Play 후 **Console**에 에러가 없는지 확인  
  - "디포머 본이 없다" → Root Bone 할당 또는 Deform Bones 할당 확인  
  - "JellyPhysicsCore를 찾을 수 없다" → JellyBody가 JellyCharacter 자식인지 확인  

이렇게 하면 **SkinnedMesh 방식**으로 Hierarchy와 컴포넌트 설정이 끝납니다.

---

## 오브젝트 계층 구조 (SkinnedMesh 방식)

```
Hierarchy
└── JellyCharacter          ← Rigidbody, CapsuleCollider, JellyPhysicsCore, JellyGapTraversal, JellyCharacterController
    └── JellyBody            ← SkinnedMeshRenderer, JellySkinnedMeshDeformer (리깅된 메시만 사용)
```

- JellyCharacter에는 **MeshFilter / MeshRenderer를 넣지 않습니다.**
- JellyBody에는 **Skinned Mesh Renderer**와 **JellySkinnedMeshDeformer**만 넣습니다. (Mesh 방식 컴포넌트는 사용하지 않음)

## JellyBody (자식) — SkinnedMesh 방식

리깅된 캐릭터(스킨 메시)를 쓰는 구성입니다. **MeshFilter / MeshRenderer / JellyMeshDeformer는 사용하지 않습니다.**

| 컴포넌트 | 비고 |
|----------|------|
| Transform | 기본 |
| SkinnedMeshRenderer | ⚠️ Mesh, Root Bone, Bones 설정 완료된 스킨 메시 |
| JellySkinnedMeshDeformer | 스크립트 추가 |

### JellyBody Transform 설정 (SkinnedMesh 방식)
| 항목 | 값 |
|------|-----|
| Position | (0, 0.5, 0) → CapsuleCollider 중심과 맞추기 |
| Rotation | (0, 0, 0) |
| Scale | (1, 1, 1) 권장 (본/메시 스케일은 에셋 기준 유지) |

### SkinnedMeshRenderer 설정
- **Mesh**: 리깅된 메시 에셋 할당
- **Root Bone**: 해당 스킨의 루트 본 할당
- **Bones**: 자동 바인딩되어 있으면 그대로 사용
- **Materials**: Element 0에 머티리얼 할당 (URP Lit 등)

### JellySkinnedMeshDeformer 설정
| 항목 | 값 |
|------|-----|
| Deform Bones | (선택) 변형을 줄 본. 비어 있으면 SkinnedMeshRenderer의 Root Bone만 사용 |
| Deform Multiplier | 1.5 |
| Deform Smoothing | 0.08 |
| Squash Stretch Ratio | 0.8 |
| Preserve Volume | ✅ |

- **Deform Bones**가 비어 있으면 스크립트가 **SkinnedMeshRenderer → Root Bone**을 자동 사용합니다.
- 여러 본에 변형을 주고 싶으면 해당 본들을 Deform Bones에 넣어 주세요 (예: Spine, Chest 등). 단, 부모-자식 관계인 본을 모두 넣으면 스케일이 중복 적용되므로, 보통은 **루트 본 하나만** 사용하는 것을 권장합니다.

## 최종 체크리스트 (SkinnedMesh 방식)

**Play 전:**
- [ ] JellyCharacter에 MeshFilter/MeshRenderer **없는가** (있으면 제거)
- [ ] JellyBody에 **Skinned Mesh Renderer** + **JellySkinnedMeshDeformer**만 있는가 (MeshFilter, JellyMeshDeformer 없어야 함)
- [ ] JellyBody **Skinned Mesh Renderer** → Mesh, Root Bone, Materials 할당되어 있는가
- [ ] JellyGapTraversal → **Gap Wall Layer**: Wall 체크
- [ ] JellyCharacterController → **Ground Layer**: Ground 체크
- [ ] 바닥 오브젝트 Layer: **Ground**, 벽 오브젝트 Layer: **Wall**

**Play 후 Console 에러 확인:**
- "JellyPhysicsCore를 찾을 수 없다" → JellyBody가 JellyCharacter의 **자식**인지 확인
- "디포머 본이 없다" → Skinned Mesh Renderer에 **Root Bone** 할당 또는 **Deform Bones** 할당 확인
- "SkinnedMeshRenderer에 메시가 할당되지 않았습니다" → Skinned Mesh Renderer **Mesh** 슬롯 할당 확인
